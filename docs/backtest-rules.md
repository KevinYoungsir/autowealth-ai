# A 股回测规则

## 1. 基本原则

A 股回测必须以可复现、可审计、无未来函数为基本原则。所有指标只能使用当时已经公开且可获得的数据，所有交易模拟必须符合当时市场交易约束。

回测结果仅用于研究和教育，不构成投资建议，也不代表未来表现。

## 2. 回测范围

- 市场范围：沪深京 A 股，可按阶段逐步接入。
- 时间范围：目标支持约 15 年历史回测。
- 频率：优先支持日频，后续可扩展周频、月频和分钟级数据。
- 组合频率：优先支持月度、季度再平衡，避免过度短线化。
- 基准：沪深 300、中证 500、中证 800、中证全指、创业板指、行业指数等。

## 3. 复权规则

系统应明确区分前复权、后复权和不复权价格。

### 前复权

- 用途：适合观察历史价格连续走势和技术指标研究。
- 风险：历史价格会随最新除权除息信息回溯调整，不适合作为真实交易成交价格。
- 要求：用于信号计算时必须确认不会引入未来除权信息。

### 后复权

- 用途：适合观察长期累计收益和持有回报。
- 风险：价格序列可能与真实历史成交价格差异较大。
- 要求：用于收益展示时应标明复权方式。

### 不复权

- 用途：适合模拟历史真实成交价格、涨跌停判断和交易成本计算。
- 风险：不处理分红送转会导致长期收益计算不完整。
- 要求：交易撮合、涨跌停、停牌判断优先使用不复权价格；收益计算应结合分红送转或复权因子。

## 4. 交易成本

回测必须显式设置并记录交易成本假设。

建议默认值：

- 买入佣金：成交金额的 0.03%，可配置。
- 卖出佣金：成交金额的 0.03%，可配置。
- 最低佣金：每笔 5 元，可配置。
- 印花税：卖出金额的 0.05%，按当前或历史政策配置。
- 过户费：按交易所规则配置，可先作为可选项。
- 滑点：按成交金额、价格或成交量比例估算，可配置。

不同历史时期交易费用和印花税政策可能变化。严谨回测应支持按日期配置费率。

## 5. 滑点与流动性

- 默认滑点应与股票流动性、成交额、换手率和委托规模相关。
- 不允许假设任意规模都能按收盘价无成本成交。
- 调仓成交额不应超过当日成交额的一定比例。
- 对低流动性股票应提高滑点或限制买入。
- 大额组合回测应使用容量约束和冲击成本估计。

## 6. 涨跌停、停牌和可交易性

### 涨跌停

- 当日涨停时，买入可能无法成交，应根据撮合规则限制买入。
- 当日跌停时，卖出可能无法成交，应根据撮合规则限制卖出。
- 不同板块、ST 状态和历史阶段的涨跌停幅度不同，必须按当时规则处理。

### 停牌

- 停牌期间不能成交。
- 停牌股票价格和市值处理方式必须固定，例如沿用前收盘价估值。
- 停牌导致无法调仓时，应记录未成交订单和实际偏离权重。

### 可交易性

- 新股上市初期、临停、长期停牌、流动性不足、重大异常状态等应有可配置过滤规则。
- 调仓日不可交易的股票不能被视为按目标权重成功买入或卖出。

## 7. ST、退市和风险状态

- ST、*ST、退市整理、暂停上市、终止上市等状态必须按历史时间点记录。
- 股票进入风险状态后是否保留、卖出或禁止买入，应作为策略参数明确。
- 退市股票不能从历史股票池中删除，否则会产生幸存者偏差。
- 退市处理必须保守，例如按最后可交易价格、退市整理期价格或归零情景进行压力测试。

## 8. 财报发布时间滞后

财务因子必须按财报真实披露日期进入回测，不能按报告期结束日直接使用。

要求：

- 使用公告披露日或可获得日作为财务数据生效日期。
- 一季报、半年报、三季报和年报均需处理披露滞后。
- 修正公告、业绩预告、快报和正式财报必须区分数据口径。
- 调仓时点只能使用当日之前已经公开的数据。

## 9. 幸存者偏差规避

- 历史股票池必须包含当时上市且可交易的全部股票，包括后来退市或长期停牌的股票。
- 指数成分股回测必须使用历史成分，不能使用当前成分回填历史。
- 行业分类、公司名称、ST 状态和上市状态必须按历史版本处理。
- 缺失数据不能简单删除样本，应记录缺失原因并采用固定规则处理。

## 10. 未来函数和数据泄露规避

禁止以下行为：

- 使用未来价格、未来收益或未来成交量生成当期信号。
- 使用未来披露的财报数据计算历史调仓信号。
- 使用全样本标准化、全样本分位数或未来窗口统计。
- 使用当前指数成分、当前行业分类或当前存续股票池回测过去。
- 在训练机器学习模型时让测试期信息进入训练、标准化或特征选择流程。

要求：

- 所有滚动指标必须只使用当前时点之前的数据。
- 因子标准化、行业中性化和分位数计算必须在截面当期完成。
- 训练、验证、测试必须按时间切分，避免随机打乱导致泄露。
- 回测日志应记录每次调仓可用数据快照或数据版本。

## 11. 组合级回测指标

回测报告至少包含以下组合级指标：

- 年化收益：组合在回测期内的年化复合收益率。
- 最大回撤：组合净值从峰值到谷值的最大跌幅。
- 夏普比率：超额收益相对波动率的风险调整指标。
- 卡玛比率：年化收益相对最大回撤的指标。
- 波动率：组合收益率的年化标准差。
- 换手率：组合调仓导致的交易比例，可按单期和年化展示。
- 年度收益：按自然年统计组合收益。
- 月度收益：按自然月统计组合收益。

建议补充指标：

- Alpha、Beta、信息比率、跟踪误差。
- 胜率、盈亏比、最长回撤持续期。
- 行业暴露、风格暴露、个股集中度。
- 成本前收益与成本后收益对比。

## 12. 回测输出要求

每次回测必须记录：

- 数据源、数据版本、拉取时间和缓存路径。
- 回测区间、股票池规则、基准、调仓频率。
- 复权方式、成交价格规则和费用假设。
- 不可成交、停牌、涨跌停、退市处理记录。
- 策略参数、因子版本和风控约束。
- 指标结果、风险提示和失效情景。

## 13. 第三阶段组合级回测实现

第三阶段新增独立的组合级研究回测模块，位置为 `autowealth/backtest/`。本阶段只实现多标的、多权重、定期调仓、交易成本和基础绩效指标，不实现选股策略、不接入 DeepSeek、不开发前端看板、不接入真实交易接口。

新增模块：

- `autowealth/backtest/portfolio_backtester.py`：`PortfolioBacktester`，支持多只 A 股组合回测、目标权重、现金仓位、单股最大权重、手续费、印花税和滑点。
- `autowealth/backtest/metrics.py`：计算日收益、总收益、年化收益、最大回撤、波动率、夏普比率、卡玛比率、年度收益和月度收益。
- `autowealth/backtest/rebalance.py`：生成调仓日期，支持月度、季度、年度和五年调仓。

当前阶段输出字段：

- `equity_curve`
- `daily_returns`
- `annualized_return`
- `total_return`
- `max_drawdown`
- `volatility`
- `sharpe_ratio`
- `calmar_ratio`
- `turnover`
- `trade_log`
- `holdings_by_period`
- `annual_returns`
- `monthly_returns`

重要边界：

- 输入权重是外部给定的目标权重，不代表系统生成的选股策略。
- 权重总和不能超过 1；未分配部分作为现金保留。
- `cash_weight` 是最低现金仓位约束，目标股票权重加现金仓位不能超过 1。
- `max_position_weight` 用于限制单只股票最大目标权重。
- 手续费、印花税和滑点是研究假设，不代表任何券商或真实市场成交承诺。
- 当前实现使用收盘价进行研究性调仓模拟，未模拟涨跌停、停牌、部分成交、流动性容量和真实撮合。
- 回测结果仅用于研究和教育，不构成投资建议，也不代表未来表现。

## 14. 组合回测使用示例

使用外部已准备好的统一行情数据：

```python
from autowealth.backtest import PortfolioBacktester

target_weights = {
    "600519": 0.2,
    "000001": 0.2,
    "600036": 0.2,
    "600900": 0.2,
    "000858": 0.2,
}

backtester = PortfolioBacktester(
    initial_capital=1_000_000,
    start_date="2009-01-01",
    end_date="2024-12-31",
    rebalance_frequency="yearly",
    commission=0.0003,
    stamp_tax=0.0005,
    slippage=0.0002,
    cash_weight=0.0,
    max_position_weight=0.3,
)

result = backtester.run(target_weights, price_data=price_data)
```

使用第二阶段的 A 股数据层按需只读拉取：

```python
from autowealth.backtest import PortfolioBacktester
from autowealth.data import AShareDataProvider

backtester = PortfolioBacktester(
    data_provider=AShareDataProvider(),
    initial_capital=1_000_000,
    start_date="2009-01-01",
    end_date="2024-12-31",
    rebalance_frequency="quarterly",
    max_position_weight=0.25,
)

result = backtester.run(
    {
        "600519": 0.2,
        "000001": 0.2,
        "600036": 0.2,
        "600900": 0.2,
        "000858": 0.1,
    },
    adjust="qfq",
)
```

示例中的标的和权重只用于说明接口形态，不构成投资建议。

## 15. 第十二阶段时点权重调度

`PortfolioBacktester.run_weight_schedule` 是现有 `run` 的兼容扩展，不改变原有固定目标权重入口。真实研究流水线按每个年度或五年调仓日提供独立目标权重映射，未入选标的在该期显式设为 0，以便退出旧持仓。

每期目标权重只能由 execution date 前的 signal date 当时可获得的数据生成。
signal date 是严格早于 execution date、且所有已加载组合标的都有真实行情的
最近对齐交易日。价格输入限定为 `date <= signal_date`，基本面同时满足
`report_date <= signal_date` 与 `available_date <= signal_date`，宏观数据也按
signal date 做 as-of 选择。

当前兼容入口仍使用已有收盘价成交与成本模型。execution date 当日先按旧
持仓计算收盘净值，再在该收盘价执行调仓；新权重从该收盘价之后生效，
因此不会取得 execution 当日已经发生的价格变化。权重调度中的日期是明确
execution dates，不再由 backtester 二次生成。

价格矩阵最多前向填充 5 个组合交易日用于估值对齐，禁止无限沿用陈旧价格。
真实流水线只在所有已加载标的具有 execution date 真实 bar 时生成调仓计划；
缺失 bar、零成交量等情况继续形成可交易性 warning。停牌、涨跌停、ST、
退市、成交容量和公司行动尚未被完整执行级建模，不能据此声称为严格可成交
回测。完整限制见 `docs/real-data-research.md`。

## 16. 年度收益首年口径

`annual_returns` 的首个有效年度使用该年度首个有效净值到年末净值计算，
因此从年初附近开始的 2024–2025 两年曲线会同时包含 2024 和 2025。
后续年度使用相邻年末净值计算。某年度只有一个有效净值点时无法计算，
该年度不进入年度收益，并由真实研究 metrics 的 `excluded_partial_years`
显式记录；`annual_return_method` 记录计算口径。
