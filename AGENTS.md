# Codex 开发规则

本项目将逐步改造成面向 A 股长期投资组合研究的系统。后续所有 Codex 协作都必须遵守本文档，优先保证研究严谨性、风险表述合规性和代码演进可审计性。

## 项目定位

- 本项目仅用于投资研究、教育和策略验证，不构成任何投资建议、收益承诺或交易指令。
- 系统目标是辅助研究长期投资组合，不是自动交易系统。
- 任何输出都不得直接或间接承诺确定收益、保本、固定回报或低风险高收益。
- 年化收益目标只能表达为研究目标区间或历史回测目标，例如“目标长期年化收益区间 10%-15%”，必须同时说明历史回测不代表未来表现。

## 开发流程

- 所有改动必须小步提交，单次改动聚焦一个明确主题。
- 每次修改前先阅读相关模块、测试和文档，不做脱离上下文的重写。
- 第一阶段仅新增项目规划和开发规范，不实现 A 股业务代码。
- 不得修改与当前任务无关的文件，不得回滚用户已有改动。
- 新增代码前必须先补充或更新对应的规则、接口说明或测试计划。
- 涉及回测、数据源、因子、风控、报告生成的改动，应在文档中说明假设、边界和失效场景。

## 投资与合规边界

- 不允许写入真实交易接口、自动下单逻辑、券商交易 API 对接或实盘资金操作功能。
- 不允许保存、生成或提交真实 API Key、账号、Token、Cookie、证书或私钥。
- 不允许把模型输出描述为“买入/卖出指令”，只能描述为研究信号、候选清单、风险提示或复核结论。
- 所有投资分析、组合构建和回测结果必须附带风险提示：仅用于研究和教育，不构成投资建议。
- 对任何收益、回撤、胜率、夏普、卡玛等指标的描述都必须标明数据区间、假设条件和限制。

## 回测严谨性要求

- 回测必须避免未来函数、幸存者偏差、数据泄露和财报发布时间错配。
- 财务数据、公告数据、指数成分、ST 状态、退市状态等必须按当时可获得时间点使用。
- 组合调仓只能使用调仓时点之前已经公开且可获得的数据。
- 回测必须显式处理手续费、印花税、滑点、停牌、涨跌停、ST、退市和流动性约束。
- 回测结果必须输出组合级指标，至少包括年化收益、最大回撤、夏普比率、卡玛比率、波动率、换手率、年度收益和月度收益。
- 不得只使用当前仍存续股票构造历史股票池；必须考虑历史上市、退市、停复牌和名称状态变化。

## DeepSeek 与大模型使用边界

- DeepSeek 或其他大模型只能用于研究摘要、数据异常提示、风险复核、反方观点生成和报告润色。
- 大模型不能直接决定买卖、仓位、调仓或交易执行。
- 大模型输出必须可追溯到结构化数据、规则引擎或人工可审查的研究证据。
- 大模型结论必须标记为“辅助研究意见”，不得作为确定性判断。

## 数据与缓存规范

- A 股行情、指数、财务、宏观、新闻和公告数据应优先通过可复现的数据管线接入。
- 优先考虑 AKShare、东方财富和本地 parquet 缓存；后续可扩展 Tushare、BaoStock、JQData、Wind、Choice。
- 本地缓存应记录数据源、拉取时间、覆盖区间、复权方式、字段版本和校验摘要。
- 数据清洗逻辑必须可测试、可复现，并保留异常样本处理规则。

## 文档和测试要求

- 新增或修改策略逻辑前，应更新 `docs/strategy-spec.md`。
- 新增或修改回测逻辑前，应更新 `docs/backtest-rules.md`。
- 新增或修改数据源前，应更新 `docs/data-source-plan.md`。
- 重要研究假设必须写入文档，不得只存在于代码注释或临时对话中。
- 后续实现代码时，必须补充针对偏差规避、边界条件和指标计算的测试。
